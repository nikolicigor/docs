## List Pipeline Runs

- Method: GET
- Path: `/pipeline-runs`
- Auth: `X-API-Key`

Query params:

| Name | Type | Description |
|------|------|-------------|
| page | number | Page number |
| limit | number | Page size |
| clientId | string | Filter by client ID |
| pipelineId | string | Filter by pipeline ID |
| status | string | pending, processing, completed, failed, acknowledged |
| model | string | Schema/pipeline name filter |
| clientName | string | Filter by client name |
| id | string[] | Filter by specific IDs |

```bash
curl -H "X-API-Key: $FILEFEED_API_KEY" \
  "https://api.filefeed.io/pipeline-runs?status=completed&limit=50"
```

## Get a Pipeline Run

- Method: GET
- Path: `/pipeline-runs/{id}`

```bash
curl -H "X-API-Key: $FILEFEED_API_KEY" \
  "https://api.filefeed.io/pipeline-runs/run_123"
```

## Get Run Data (Paginated)

- Method: GET
- Path: `/files/pipeline-runs/{id}`
- Query: `offset`, `limit`

```bash
curl -H "X-API-Key: $FILEFEED_API_KEY" \
  "https://api.filefeed.io/files/pipeline-runs/run_123?offset=0&limit=1000"
```

## Acknowledge a Run

- Method: PATCH
- Path: `/pipeline-runs/{id}/status`
- Body: `{ "status": "acknowledged" }`

```bash
curl -X PATCH -H "X-API-Key: $FILEFEED_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"status":"acknowledged"}' \
  "https://api.filefeed.io/pipeline-runs/run_123/status"
```

## Reprocess a Run

- Method: POST
- Path: `/files/pipeline-runs/{id}/reprocess`

```bash
curl -X POST -H "X-API-Key: $FILEFEED_API_KEY" \
  "https://api.filefeed.io/files/pipeline-runs/run_123/reprocess"
```

## Presigned URLs

- Original file: `GET /files/pipeline-runs/{id}/original?expiresIn=3600`
- Processed file: `GET /files/pipeline-runs/{id}/processed?expiresIn=3600`

## Runs by Pipeline

- Method: GET
- Path: `/pipeline-runs/pipeline/{pipelineId}`
- Query: `page`, `limit`

Use the SDK for typed responses and pagination helpers.
